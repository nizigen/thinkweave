# APP_FLOW.md — 用户流程文档

## 核心用户旅程

### 旅程1：创建并执行长文本生成任务

```
[首页/任务创建页]
    │
    ├── 用户输入：任务主题（如"写一篇关于量子计算的技术报告"）
    ├── 选择生成模式：技术报告 / 小说 / 自定义
    ├── 选择研究深度：快速（粗粒度DAG）/ 标准（默认）/ 深入（细粒度DAG，更多子任务）
    ├── 可选：指定字数目标（默认10000字）
    ├── 可选：指定风格要求（学术/科普/叙事）
    ├── 可选：自定义模式下可自由描述结构偏好
    │
    └── 点击「开始生成」
            │
            ▼
[系统后台处理]
    ├── 编排层Agent：调用LLM分解任务 → 生成子任务DAG
    ├── 返回：任务ID + DAG结构预览
    │
    └── 跳转至「DAG预览确认页」（自动带任务ID）

[DAG预览确认页]（嵌入监控页顶部或独立弹窗）
    ├── 展示LLM分解出的DAG结构（节点树形图）
    ├── 每个节点显示：子任务名称 / 预估耗时 / 分配Agent角色
    ├── 用户操作：
    │   ├── 点击「确认执行」→ 正式启动任务调度，跳转「编排监控页」
    │   └── 点击「手动调整」→ 进入DAG编辑模式（可增删节点、调整依赖关系）
    │           └── 调整完成 → 点击「确认执行」→ 以修改后的DAG正式启动

[编排监控页]
    ├── 左侧：实时DAG图（节点颜色随状态变化）
    │   └── 节点可点击 → 右侧展开节点详情面板：
    │           ├── 当前输入内容
    │           ├── 当前输出内容（流式）
    │           ├── 已耗时 / 预计剩余时间
    │           └── Token消耗（输入/输出分开展示）
    ├── 右侧上：Agent活动面板（谁在干什么）
    ├── 右侧下：执行日志实时滚动
    ├── 顶部：长文本生成阶段进度条（大纲→写作→审查→一致性→完成）
    ├── 右侧实时预览面板（长文本模式）：
    │   └── 实时显示已生成的Markdown内容（章节完成即追加）
    │
    ├── 手动干预工具栏：
    │   ├── 「暂停执行」→ 当前Agent完成当前步骤后暂停，等待用户操作
    │   ├── 「跳过当前节点」→ 将当前节点标记为已跳过，调度器继续执行后续节点
    │   └── 「手动重试」→ 对选中的失败/暂停节点重新触发执行
    │
    ├── 执行过程中：
    │   ├── 大纲Agent完成 → 大纲节点变绿 → 触发写作Agent（并行）
    │   ├── 各章节写作完成 → 逐节点变绿
    │   ├── 审查Agent完成 → 若发现问题 → 对应节点变黄 → 重新写作
    │   └── 一致性Agent完成 → 全部节点变绿
    │
    └── 任务完成 → 显示「查看结果」按钮

[结果展示页]
    ├── 左侧：Markdown渲染的完整文档
    ├── 右侧：文档目录（章节锚点导航）
    ├── 顶部操作栏：
    │   ├── 导出为 DOCX
    │   ├── 导出为 PDF
    │   └── 保存到历史
    │
    └── 底部：本次生成统计（字数/耗时/Token消耗/参与Agent数）
```

### 旅程2：Agent管理

```
[导航栏] → 点击「Agent管理」

[Agent管理页]
    ├── 查看当前所有Agent（表格：名称/角色/层级/状态/当前任务）
    ├── 点击「注册新Agent」
    │   ├── 填写：名称、角色类型（规划/写作/审查/工具）、层级（0/1/2）
    │   ├── 填写：能力描述（自然语言，用于LLM分配任务时参考）
    │   ├── 选择：对接的LLM模型（GPT-4 / DeepSeek-V3 / 自定义）
    │   └── 提交 → Agent注册成功，状态为「空闲」
    │
    ├── 点击某Agent → 查看详情
    │   ├── 历史任务列表
    │   ├── 执行成功率
    │   └── 最后活跃时间
    │
    └── 点击「停用」→ Agent下线，正在执行的任务转移到备用Agent
```

### 旅程4：长文本生成专用流程（详细子流程）

```
[任务创建页 - 长文本模式]
    ├── 输入主题 + 选择模式（技术报告 / 小说）
    ├── 选择研究深度：快速（3000字）/ 标准（10000字）/ 深入（20000字）
    ├── 可选：指定章节结构偏好（如"分为6章，第1章为背景，第2章为现状..."）
    └── 点击「开始生成」
            │
            ▼
[大纲生成阶段]
    ├── 大纲Agent调用LLM生成章节结构（Markdown格式）
    ├── 跳转「大纲编辑页」（/task/:taskId/outline）
    │   ├── 在线编辑器展示大纲（支持直接编辑章节标题/层级/顺序）
    │   ├── 用户可增删章节、调整层级
    │   └── 点击「确认大纲」→ 保存大纲并进入写作阶段
    │
            ▼
[写作阶段]
    ├── 调度器根据确认大纲创建写作任务DAG
    ├── ★ Session Memory 初始化：为本次任务创建独立记忆命名空间
    ├── ★ 大纲中的章节领地分配（topic_claims）写入 Session Memory
    ├── 多个写作Agent并行生成各章节内容
    ├── 每个写作Agent启动前：
    │   ├── 读取 Session Memory：其他已完成章节的摘要 + 已用图片/引用
    │   ├── 读取本章 topic_claims：明确 owns（负责的论点）和 boundary（不覆盖的内容）
    │   └── 将去重指令注入 Writer prompt（"以下内容已被其他章节覆盖，请勿重复"）
    ├── 每个写作Agent完成后：
    │   └── ★ 将章节摘要（key_points + word_count + used_evidence）写入 Session Memory
    ├── 监控页实时预览面板追加已完成章节内容
    └── 全部章节写作完成 → 自动进入审查阶段
            │
            ▼
[审查阶段]
    ├── 审查Agent对每章独立评分（0-100分）
    │   ├── ★ 审查前：从 Session Memory 读取内容重叠检测结果（向量相似度 > 0.85 的章节对）
    │   ├── 评分 >= 70 且无严重重叠：章节通过，节点变绿
    │   ├── 评分 < 70 或检测到重叠：章节不通过，节点变黄 → 自动触发重写（最多3次）
    │   └── 重写指令中包含具体的去重要求（"与第X章内容重叠，请改写以下段落"）
    ├── 用户可在监控页点击审查节点查看详细评审报告（扣分项/改进建议/重叠检测结果）
    └── 全部章节通过 → 进入一致性检查阶段
            │
            ▼
[一致性检查阶段]
    ├── 一致性Agent全文扫描
    │   ├── ★ 从 Session Memory 读取所有章节摘要（无需全文，降低 token 消耗）
    │   ├── 检测风格不一致（语气、用词、格式）
    │   └── 检测逻辑矛盾（前后章节内容冲突、重复内容）
    ├── 生成问题清单，标明涉及的章节和具体问题
    │   ├── 无问题 → 进入「记忆清理 + 知识提升」→ 跳转「结果展示页」
    │   └── 有问题 → 将问题清单发给对应写作Agent修改
    │           ├── 写作Agent根据问题清单定向修改
    │           └── 修改完成 → 一致性Agent再次检查（最多循环2次）
    ├── ★ 任务完成时：Session Memory 清理
    │   ├── 将高价值数据提升到 Knowledge Graph（已验证引用、实体关系、术语定义）
    │   └── 删除临时 Session 命名空间
    └── 一致性检查通过 → 跳转「结果展示页」
            │
            ▼
[结果展示页]
    ├── 左侧：Markdown渲染的完整文档
    ├── 右侧：文档目录（章节锚点导航）
    ├── 顶部操作栏：导出DOCX / 导出PDF / 保存到历史
    └── 底部：生成统计（字数/耗时/Token消耗/参与Agent数）
```

## 错误处理流程

| 场景 | 处理方式 |
|------|----------|
| LLM API超时 | 自动重试3次，超过后标记节点为「失败」，提示用户手动重试 |
| Agent执行失败 | 任务转移到同层级备用Agent，监控页显示橙色警告 |
| Redis连接断开 | 前端显示「连接中断」横幅，后端自动重连 |
| 生成内容不一致（审查不通过） | 对应章节节点变黄，自动触发重写，最多3次 |
| 用户主动终止 | 点击「停止」→ 发送取消信号 → 已完成节点保留，未完成标记取消 |
| LLM返回格式不符合JSON Schema | 自动修复重试（最多2次）；仍失败则切换备用模型重试 |
| Agent响应超时（>60s） | 节点标记超时，调度器自动将任务转移给新Agent实例重新执行 |
| WebSocket连接断开 | 前端使用指数退避自动重连（1s → 2s → 4s → ...）；重连成功后自动同步最新任务状态 |

## 5. 参考仓库流程增强（2026-03-21）

### 5.1 增加两个强制关卡
1. Stage 2.5：`PRE_REVIEW_INTEGRITY`
   - 位置：写作完成后、进入审查前。
   - 失败处理：进入“修复并重验”循环（最多 3 次）。
2. Stage 4.5：`FINAL_INTEGRITY`
   - 位置：复审/再修订后、最终导出前。
   - 失败处理：禁止 finalize，必须回修订。

### 5.2 交互检查点（Checkpoint）
1. `FULL`：展示完整阶段产物、质量指标、下一步选项。
2. `SLIM`：仅展示一句状态，适用于连续自动继续场景。
3. `MANDATORY`：完整性失败、评审决策、最终导出前，必须用户确认。

### 5.3 中途接入流程
1. 检测用户输入材料类型（仅题目/有初稿/有评审意见）。
2. 自动推导入口阶段并提示确认。
3. 自动补齐不可跳过阶段（至少一次完整性校验）。
4. 入口检测结果写入任务元数据，供 UI 与日志统一展示。

## 2026-03-21 Flow Addendum — Retry/Fallback Execution Path
During node execution:
1. Scheduler sends node payload with `model` + `agent_config`.
2. Agent runtime resolves call params.
3. LLM call executes retry policy from `max_retries`.
4. On primary failure, fallback order uses `fallback_models` (if provided), else model default fallback.
5. Failure event emitted only after chain exhaustion.
