# Dan Koe Content Perpetual Motion Machine (English)

## System Overview

A skill system based on Dan Koe's content creation methodology, helping creators transition from "inspiration-driven" to "system-driven" production.

**Core philosophy:**
- Don't write more — squeeze every drop from what you've already written
- A good idea should only cost you brainpower once
- You don't lack content — you just haven't fully used what you have

## Four Core Skills

### Skill 1: Content Structure Judge (content-structure-judge)

**Purpose:** Determine whether content is worth repurposing

**When to use:**
- You have a high-performing piece of content to analyze
- Building a content swipe file
- Understanding the underlying structure of successful content

**Output:**
- 8-dimension structural analysis
- Reusable expression templates
- Clear reuse verdict

**Details:** `content-structure-judge/SKILL.md`

---

### Skill 2: Pre-Writing Clarifier (pre-writing-clarifier)

**Purpose:** Force clarity on key decisions before writing

**When to use:**
- Staring at a blank page with no idea where to start
- Lots of ideas but disorganized logic
- Writing in circles without progress

**Output:**
- 6 mandatory questions answered
- Clear writing blueprint
- Defined creative direction

**Details:** `pre-writing-clarifier/SKILL.md`

---

### Skill 3: Mother Content Architect (mother-content-architect)

**Purpose:** Upgrade a validated idea into long-term reusable core content

**When to use:**
- A short-form post performed noticeably well
- An idea keeps getting quoted and commented on
- You sense a particular idea is important

**Output:**
- Complete mother content blueprint
- 5-step construction process
- Human vs AI division of labor

**Details:** `mother-content-architect/SKILL.md`

---

### Skill 4: Content Repurposing Engine (content-repurposing-engine)

**Purpose:** Think once, distribute many times

**When to use:**
- Just finished a long-form article
- Recorded a full video/podcast
- Have a validated core idea

**Output:**
- 20+ short-form pieces
- Multi-platform adapted versions
- Complete video script
- CTA options library

**Details:** `content-repurposing-engine/SKILL.md`

## Correct Usage Order (Critical!)

**Wrong order = broken system**

### The only correct workflow:

```
1. Analyze (content-structure-judge)
   |
   Analyze successful content, extract reusable structures

2. Clarify (pre-writing-clarifier)
   |
   Define writing direction, complete 6 key decisions

3. Write once (mother-content-architect)
   |
   Build mother content, create a content asset

4. Maximize (content-repurposing-engine)
   |
   Repurpose and distribute, achieve content compounding
```

## Quick Start Guide

### First Time Use

**Step 1: Build your swipe file**
```
Collect 10-15 of your top-performing pieces
-> Use content-structure-judge to analyze each one
-> Build a library of reusable structures
```

**Step 2: Create your first mother content**
```
Choose a validated idea
-> Use pre-writing-clarifier to define direction
-> Use mother-content-architect to build structure
-> Complete a 2000-5000 word mother content piece
```

**Step 3: Start repurposing**
```
After mother content is complete
-> Use content-repurposing-engine to repurpose
-> Get 20+ publishable pieces
-> Create a 2-4 week publishing schedule
```

### Ongoing Use

**Daily workflow:**
- Create 1 mother content piece per week
- Publish 2-3 repurposed pieces per day
- Analyze data once per month, optimize structure library

**Monthly review:**
- Track which structures perform best
- Update swipe file
- Retire ineffective patterns

## Success Indicators

When you truly master these 4 Skills, you'll notice:

**No more topic anxiety**
- Your swipe file has plenty of structural templates
- You know what types of content work

**3x writing efficiency**
- Clear direction means no more endless revisions
- Clear structure means smooth writing flow

**Consistent output**
- 1 mother content piece sustains 2-4 weeks of publishing
- No dependence on inspiration — systematic production

**Growing influence**
- Core ideas get repeatedly shared
- You build a personal content brand

## Common Mistakes

| Mistake | Consequence | Correct Approach |
|---------|-------------|-----------------|
| Skip analysis, write directly | Don't know what structures work | Build swipe file first |
| Skip clarification, think while writing | Unclear direction, constant rework | Complete 6 questions first |
| Let AI write everything | Content lacks personal voice | Human decides, AI assists |
| Publish once, don't repurpose | Waste content value | Think once, use many times |

## Advanced Techniques

### Building a Content Asset Library

**Three-layer structure:**
1. **Structure library**: Validated content frameworks (from Skill 1)
2. **Mother content library**: Core long-form pieces (from Skill 3)
3. **Repurposed library**: Ready-to-publish short-form content (from Skill 4)

**Management tools:**
- Notion / Obsidian: Structured management
- Spreadsheets: Track performance data
- Note apps: Quick idea capture

### Team Collaboration

**Role division:**
- Creator: Responsible for Skills 1, 2, 3 (core decisions)
- AI assistant: Supports Skills 3, 4 (execution details)
- Operations: Handles Skill 4 (distribution scheduling)

### Data-Driven Optimization

**Metrics to track:**
- Structure library: Which frameworks perform best
- Mother content: Which topics get highest engagement
- Repurposed content: Which platform delivers best results

**Optimization loop:**
```
Publish -> Data analysis -> Structure optimization -> Republish
```

## Integration with Other Tools
