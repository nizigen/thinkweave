## Tasks

<task type="code">
  <name>Researcher tool execution path</name>
  <files>backend/app/agents/worker.py, backend/app/agents/researcher_agent.py, backend/app/utils/llm_client.py</files>
  <action>为 researcher 角色引入受控 `chat_with_tools` 执行路径（report 模式默认开启），支持 web/mcp 检索并返回结构化证据。</action>
  <verify>researcher 节点运行日志中出现工具调用轨迹；失败时具备可解释降级。</verify>
  <done>researcher 不再只输出计划，而是输出含来源元数据的 evidence ledger。</done>
</task>

<task type="code">
  <name>Evidence ledger contract hardening</name>
  <files>backend/app/prompts/researcher/research.md, backend/app/prompts/writer/write_chapter.md, backend/app/prompts/reviewer/review_chapter.md, backend/app/schemas/task.py</files>
  <action>统一 evidence 数据契约（source_url/title/published_at/claim_binding/confidence），writer/reviewer 强制消费证据账本并回填 citation mapping。</action>
  <verify>prompt contract tests + schema tests 通过；非法/缺字段输出被拦截并修复。</verify>
  <done>任务详情可追踪章节级 claim -> evidence -> source 链路。</done>
</task>

<task type="code">
  <name>API observability for citations</name>
  <files>backend/app/services/task_service.py, backend/app/schemas/task.py</files>
  <action>在任务详情中新增 citation/evidence 摘要字段（覆盖率、未绑定 claim 数、来源分布），用于监控和验收。</action>
  <verify>task detail API 返回稳定字段且前向兼容旧调用方。</verify>
  <done>monitor 与 e2e 可直接读取证据质量指标。</done>
</task>
