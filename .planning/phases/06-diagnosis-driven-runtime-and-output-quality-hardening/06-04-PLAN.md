# Phase 6 Plan 04: Quantified Specificity, Actionability, And Feedback Loop

<objective>
修复反例边界模糊、实施路径不可操作、建议未被执行的问题，形成从写作到检查再到定向返工的闭环。
</objective>

<threat_model>
- Threat: domain registry becomes stale. Mitigation: registry entries include version/source metadata and tests use fixtures, not live network.
- Threat: actionability prompts embed external pricing/training claims without evidence. Mitigation: require evidence IDs or mark estimates as assumptions.
- Threat: feedback loop causes infinite rewrites. Mitigation: semantic retry limits from Plan 01 and targeted affected-node routing.
</threat_model>
