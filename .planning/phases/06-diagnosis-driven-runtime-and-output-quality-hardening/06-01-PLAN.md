# Phase 6 Plan 01: StateStore, Locking, Retry Ownership, And Recoverable Checkpoints

<objective>
建立运行时状态的单一协调边界，解决多层状态不同步、并发覆盖、重试归属不清和 checkpoint 粗粒度的问题，为后续事件驱动和质量门禁改造提供安全基础。
</objective>

<threat_model>
- Threat: manual or automated transition writes inconsistent task/node states. Mitigation: all FSM transitions go through StateStore validation and transactional audit logging.
- Threat: concurrent Agents overwrite node output. Mitigation: TaskNode version optimistic locking and conflict errors with retry-safe routing.
- Threat: retry loops mask permanent failures. Mitigation: central RetryPolicy classifies transient/runtime failures separately from semantic gate failures.
- Threat: checkpoint contains sensitive generated content. Mitigation: store only required snapshots, hashes, and bounded context; do not add secrets to checkpoint metadata.
</threat_model>
