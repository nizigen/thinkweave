# Phase 6 Plan 05: FlowController, Event-Driven Realtime, Agent Health, And Operator Recovery

<objective>
完成诊断中的事件驱动、实时反馈、Agent 健康检查和人工干预能力，让前四个计划建立的状态与质量门禁可被用户和运维实时观察、确认和恢复。
</objective>

<threat_model>
- Threat: realtime stream exposes internal stack traces or sensitive prompt/context. Mitigation: user-facing DetailedError separates safe message from internal technical_details and redacts secrets.
- Threat: WebSocket ACK/replay duplicates state-changing actions. Mitigation: realtime messages are read-only events with idempotent message IDs.
- Threat: force-transition/skip-node can corrupt tasks. Mitigation: admin-only endpoints, StateStore validation, audit logs, and reason required.
- Threat: event-driven scheduler misses messages. Mitigation: Redis Stream consumer group, last-id persistence, replay from checkpoint, and polling fallback during migration.
</threat_model>
