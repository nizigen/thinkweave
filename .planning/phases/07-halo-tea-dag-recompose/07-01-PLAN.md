# Phase 7 Plan 01: TEA Protocol Baseline And DAG Metrics (No MCP Dependency)

<objective>
先落地 TEA 协议和 DAG 评估基线，建立“可观测 + 可回滚 + 可验证”的基础层，不改变现有调度行为。按用户要求，本计划不引入 MCP 集成。
</objective>

<threat_model>
- Threat: TEA 新协议破坏旧消息链路。Mitigation: dual-path 解码（TEA 优先，legacy fallback）+ 开关 `ENABLE_TEA_PROTOCOL=false` 可一键回退。
- Threat: 版本不兼容消息进入执行路径。Mitigation: major version hard-fail，拒绝消费并记录兼容性错误。
- Threat: 指标口径不稳定导致错误决策。Mitigation: `dag_eval.py` 先离线评估，使用固定样例回归，线上仅观测不驱动调度。
- Threat: MCP 不稳定导致进度阻塞。Mitigation: 本阶段明确不接入 MCP，不依赖 `backend/app/mcp` 运行时能力。
</threat_model>
